# AI 炒股 Agent 系统从 0 到 1 搭建路线

> **状态**：v0.1 MVP 已完成搭建。本文档记录项目从零到一的完整搭建过程，各阶段标注实际完成状态。✅ = 已完成，⚠️ = 部分完成，⬜ = 尚未开始。

## 1. 搭建目标

第一版目标不是做一个完整的量化交易平台，而是做出一个可以真实运行的本地桌面 MVP。

MVP 最终应该做到：

1. ✅ 用户可以下载并打开桌面应用。
2. ✅ 用户可以配置 AI 厂商和 API Key。
3. ✅ 用户可以记录本金、现金和持仓。
4. ✅ 用户可以在聊天框里告诉 AI 自己买了什么股票。
5. ✅ AI 可以解析用户输入，生成结构化草稿。
6. ✅ 用户确认后，系统把数据写入本地文件。
7. ✅ 首页可以显示资产卡片和股票卡片。
8. ✅ 用户可以设置 Agent 每隔一段时间运行。
9. ✅ Agent 可以读取持仓、调用行情 API、调用 AI 模型。
10. ✅ Agent 可以把分析结果写回聊天框。

## 2. 技术栈确定

### 2.1 推荐技术栈

```text
Tauri
React
TypeScript
Vite
Tailwind CSS
shadcn/ui
lucide-react
ECharts 或 Recharts
本地 JSON 文件
```

### 2.2 为什么这样选

Tauri 负责把 Web 前端打包成桌面应用。

React 负责构建界面。

TypeScript 负责定义账户、持仓、交易、Agent 任务等数据类型。

Vite 负责快速开发和构建。

Tailwind CSS 和 shadcn/ui 负责界面样式。

本地 JSON 文件负责第一版数据存储，避免让用户安装数据库。

### 2.3 暂时不做的东西

第一版先不做：

- 云端登录
- 多用户系统
- 自动交易
- MySQL / PostgreSQL
- 复杂量化回测
- 手机 App
- 券商实盘接入

```ts

loadTrades()
saveTrades()

loadAgentJobs()
saveAgentJobs()
```

### 7.3 数据写入原则

为了避免本地文件损坏：

1. 写入前先生成临时文件。
2. 写入成功后再替换正式文件。
3. 每次关键写入前创建备份。
4. JSON 解析失败时提示用户恢复备份。

### 7.4 第一版可以先用浏览器 localStorage 吗

开发原型可以用 localStorage。

正式 Tauri 版本不建议只用 localStorage，因为：

- 用户不容易备份。
- 数据目录不清晰。
- 后续迁移困难。

正式版建议通过 Tauri 文件 API 或 Rust 命令读写本地文件。

## 11. 阶段八：自然语言解析持仓 ✅ 已完成

### 11.1 解析目标

用户输入：

```text
我本金 10 万，买了 300 股贵州茅台，成本 1680，现金还剩 2 万。
```

AI 输出：

```json
{
  "intent": "update_account_and_positions",
  "account": {
    "initialCapital": 100000,
    "cashBalance": 20000
  },
  "positions": [
    {
      "name": "贵州茅台",
      "quantity": 300,
      "avgCost": 1680
    }
  ],
  "requiresConfirmation": true
}
```

### 11.2 确认机制

AI 解析后不能直接写入关键数据。

流程：

1. 用户发送自然语言。
2. AI 返回结构化草稿。
3. 系统展示确认卡片。
4. 用户点击确认。
5. 系统写入本地文件。
6. 聊天框提示已记录。

### 11.3 确认卡片内容

卡片字段：

- 操作类型
- 股票名称
- 股票代码
- 数量
- 成本价
- 涉及金额
- 对现金的影响

按钮：

- 确认写入
- 修改
- 取消

## 12. 阶段九：行情数据接入 ⚠️ 部分完成（实时报价已接入新浪财经 via Tauri 代理；K线/市场概况仍为 mock）

### 12.1 第一版行情方案

第一版需要选择一个简单稳定的数据源。

可选：

- AkShare 网关
- TuShare
- 第三方行情 API
- Yahoo Finance
- Finnhub

如果做 A 股，AkShare 很适合原型，但它通常运行在 Python 环境里。为了桌面应用简单，可以有两种方案：

方案一：使用第三方 HTTP 行情 API。

方案二：应用内置一个轻量本地行情服务，后续再做。

### 12.2 行情服务接口

统一封装：

```ts
getQuote(symbol)
getBatchQuotes(symbols)
getKline(symbol, period)
searchSymbol(keyword)
getMarketOverview()
```

### 12.3 持仓刷新

刷新流程：

1. 读取 positions.json。
2. 收集所有股票代码。
3. 调用 batch quotes。
4. 更新 currentPrice。
5. 重新计算 marketValue。
6. 重新计算 unrealizedPnl。
7. 写回 positions.json。
8. 更新首页 UI。
