# AI 炒股 Agent 系统介绍

> **版本**：v0.1 | **状态**：MVP 已实现 | **形态**：Windows 桌面 GUI 应用（Tauri + React）

## 1. 这是什么系统

AI 炒股 Agent 系统是一款面向个人投资者的本地桌面应用。

它不是传统的股票软件，也不是简单的交易记账工具，而是一个结合了“持仓记录、行情监控、AI 聊天、定时 Agent、投资记忆库”的个人投资助手。

用户下载应用后，只需要配置自己的 AI API Key 和行情数据源，就可以在本地使用。系统会记录用户本金、现金、持仓、交易、收益，并让 AI 定时分析股票走势，在聊天框里输出观点和风险提醒。

系统核心理念：

```text
用户负责决策，AI 负责记录、监控、分析和提醒。
```

系统不直接替用户下单，不承诺收益，不做自动交易。它的定位是辅助分析和投资复盘。

## 2. 这个系统能干什么

### 2.1 记录账户资产

用户可以记录：

- 本金
- 现金
- 当前总资产
- 持仓市值
- 总收益
- 收益率
- 每日资产快照

例如用户可以直接对 AI 说：

```text
我本金 10 万，现金还剩 2 万。
```

AI 会解析出账户信息，并在用户确认后写入本地数据。

### 2.2 记录股票持仓

用户可以通过自然语言告诉 AI 自己买了什么股票。

例如：

```text
我买了 300 股贵州茅台，成本价 1680。
```

系统会自动识别：

- 股票名称
- 股票代码
- 买入数量
- 成本价
- 占用资金

确认后，这只股票会出现在持仓卡片中。

### 2.3 展示资产和持仓卡片

系统首页会用卡片展示账户状态，类似一个投资工作台。

主要卡片包括：

- 总资产卡片
- 本金卡片
- 现金卡片
- 总收益卡片
- 收益率卡片
- 持仓数量卡片
- 每只股票的独立持仓卡片

每只股票卡片会展示：

- 股票名称
- 股票代码
- 持仓数量
- 成本价
- 当前价
- 市值
- 浮动盈亏
- 浮动收益率
- AI 最新判断

### 2.4 AI 聊天助手

系统右侧会有一个 AI 聊天区域，用户可以像使用 Codex 一样和 AI 对话。

用户可以问：

```text
我现在收益怎么样？
```

```text
帮我分析一下今天持仓风险。
```

```text
把比亚迪设置为重点观察。
```

```text
每隔 1 小时帮我看一次持仓。
```

AI 不只是聊天，它还可以调用系统内部工具：

- 读取账户
- 读取持仓
- 记录交易
- 查询行情
- 设置提醒
- 创建 Agent 定时任务
- 保存投资记忆

### 2.5 定时 Agent 自动巡检

这是系统最重要的功能。

用户可以设置：

```text
每隔 1 小时帮我看一次。
```

或者：

```text
每天 14:50 收盘前帮我分析一次。
```

到时间后，Agent 会自动执行：

1. 读取用户账户。
2. 读取当前持仓。
3. 调用行情 API。
4. 获取价格、涨跌幅、K 线等信息。
5. 读取用户投资偏好和历史记忆。
6. 调用 AI 模型进行分析。
7. 在聊天框输出观点。
8. 写入 Agent 执行日志。

示例输出：

```text
14:50 自动巡检完成。

当前总资产约 98,645.40 元，收益率 -1.35%。
今天主要拖累来自新能源方向，宁德时代分时走势偏弱。

我建议暂时不要急着加仓，先观察是否重新站回 5 日均线。
这不是买卖指令，只是基于当前数据的辅助分析。
```

### 2.6 投资记忆库

系统会让 AI 记住用户的投资习惯和历史观点。

例如：

- 用户偏好中长期持仓
- 用户不喜欢追高
- 用户单只股票最大仓位不超过 30%
- 用户关注新能源、白酒、AI 算力
- 某只股票当初买入的理由
- Agent 之前多次提醒过的风险

这些内容会进入记忆库，后续 Agent 分析时会自动参考。

### 2.7 风险提醒

用户可以设置条件提醒。

例如：

```text
如果贵州茅台跌破 1600，提醒我。
```

```text
如果宁德时代浮亏超过 8%，提醒我。
```

```text
如果我的总资产回撤超过 5%，提醒我。
```

系统会在 Agent 巡检时检查这些条件，一旦触发，就在聊天框和提醒中心通知用户。

## 3. 界面形态

系统界面参考 Codex 的工作方式。

整体布局：

```text
左侧：功能导航
中间：当前功能页面
右侧：AI 聊天 / Agent 对话
```

左侧功能包括：

- 总览
- 持仓
- 交易
- Agent
- 记忆
- 模型
- 数据源
- 设置

中间区域展示用户当前选择的功能页面。

右侧区域默认显示 AI 聊天。当用户进入某些专注页面，例如模型配置、Agent 设置、记忆库管理时，聊天区可以收起，让功能页面占据更多空间。

交互逻辑：

- 默认打开应用，显示总览 + 聊天。
- 点击“持仓”，中间显示持仓卡片，右侧仍可聊天。
- 点击“Agent”，中间显示定时任务和执行日志。
- 点击“模型”，进入模型配置页，聊天区可暂时收起。
- 点击聊天按钮，随时恢复 AI 助手。

## 4. 技术框架（已采用）

### 4.1 总体技术选择

已采用：

```text
Tauri + React + TypeScript + Vite + Tailwind CSS + shadcn/ui
```

### 4.2 框架分工

| 模块 | 技术 | 作用 |
| --- | --- | --- |
| 桌面应用壳 | Tauri | 打包成 Windows、macOS、Linux 桌面应用 |
| 前端框架 | React | 构建界面和交互 |
| 类型系统 | TypeScript | 保证数据结构可靠 |
| 构建工具 | Vite | 快速开发和打包前端 |
| 样式 | Tailwind CSS | 快速实现现代界面 |
| UI 组件 | shadcn/ui | 构建按钮、弹窗、卡片、表单 |
| 图标 | lucide-react | 提供统一图标 |
| 图表 | ECharts 或 Recharts | 显示收益曲线、资产变化 |
| 本地存储 | JSON 文件，后续可升级 SQLite | 保存账户、持仓、聊天和记忆 |
| API Key 存储 | 系统安全凭据 | 保存用户模型 API Key |
| Agent 调度 | React 定时器（v0.1）→ Tauri 后台任务（v0.2 计划） | 定时执行行情分析 |

### 4.3 为什么选择 Tauri

Tauri 适合这个系统的原因：

- 可以打包成本地桌面应用。
- 安装包比 Electron 更轻。
- 使用系统 WebView，资源占用较低。
- 前端仍然可以使用 React。
- 本地文件读写、系统通知、后台任务更自然。
- 用户下载后即可使用，不需要部署服务器。

### 4.4 为什么第一版不做传统后端

第一版不建议做传统 Web 后端，原因是：

- 用户只是想下载应用直接用，不想部署服务。
- 账户、持仓、API Key 都更适合本地保存。
- 做云端账户系统会增加登录、权限、部署、安全成本。
- 本地应用更适合个人投资助手。

后续如果要做云同步，可以再增加云服务。

## 5. 数据存储方式

### 5.1 v0.1：本地 JSON 文件（AppData 目录）

Windows 下数据保存在：

```text
C:\Users\用户名\AppData\Roaming\AI Stock Agent\
```

实际目录结构：

```text
config.json
account.json
positions.json
trades.json
models.json
data-sources.json
messages.json
agent-jobs.json
agent-runs.json
alerts.json
memories.json
conversations/
memories/
agent-runs/
backups/
```

前端不直接传文件路径，通过 Tauri 命令 `read_app_json` / `write_app_json` 按 key 白名单读写。

API Key 不写入 JSON，通过 `keyring` crate 存入 Windows Credential Manager。

优点：

- 用户不需要安装数据库。
- 数据容易备份。
- 数据容易导出。
- 早期开发速度快。
- 出问题时方便人工检查。

### 5.2 后续：本地 SQLite

当系统数据变多后，可以升级到 SQLite。

SQLite 对用户来说仍然不是传统数据库服务，它只是一个本地文件：

```text
app-data.sqlite
```

用户不需要安装、配置或维护数据库。

适合升级到 SQLite 的场景：

- 聊天记录很多
- Agent 执行日志很多
- 交易记录很多
- 需要复杂筛选
- 需要更可靠的数据一致性

## 7. 系统边界

系统可以：

- 帮用户记录投资信息
- 帮用户分析持仓
- 帮用户监控行情
- 帮用户提醒风险
- 帮用户复盘交易
- 帮用户管理投资规则

系统不应该：

- 自动替用户下单
- 承诺收益
- 保证涨跌
- 诱导高频交易
- 把 AI 观点包装成确定事实
- 在用户未确认时修改关键账户数据
